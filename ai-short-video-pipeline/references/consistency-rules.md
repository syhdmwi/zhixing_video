# Consistency Rules

这个参考文件用于控制“同一人物、同一风格、同一条视频”的稳定性。

## 1. Build Three Immutable Blocks First

在写任意单镜头提示词之前，先固定以下三块：

### Character Block

至少写清：

- 性别表现
- 年龄段
- 脸型
- 五官特征
- 发型与发色
- 身材与身高感
- 主服装
- 常驻配饰
- 身份气质

如果有真人参考图，优先基于参考图归纳，而不是凭空扩写。

### Style Block

至少写清：

- 画面质感，例如电影感、广告写真、纪实抓拍、插画
- 光线类型
- 色彩倾向
- 镜头语言
- 景别偏好
- 时代/空间背景
- 画面清晰度与细节级别

### Negative Block

至少约束：

- 人脸漂移
- 发型或服装漂移
- 多手指、肢体畸形
- 五官不对称
- 多余人物入镜
- 背景主体抢戏
- 低清晰度、糊脸、局部崩坏

## 2. Keep Global Blocks Stable Across All Shots

单镜头提示词中允许变化的是：

- 动作
- 机位
- 场景
- 道具
- 表情
- 前后景关系

默认不应变化的是：

- 主角身份
- 核心脸部特征
- 发型和主服装
- 总体风格
- 色彩语言

## 3. Prefer Reference Anchors When Available

如果模型支持参考图、一致性角色、固定 seed 或风格锁定，优先使用。这个 skill 不假设具体参数名，但应该在输出中提醒用户：

- 同一角色尽量复用同一参考图组
- 同一批次尽量复用同一风格基底
- 先通过静帧确认一致，再批量扩镜头

## 4. Reduce Variation Per Shot

一个镜头建议只引入一到两个变量变化，例如：

- 只换机位，不换服装和时代背景
- 只换动作，不换人物身份和场景基调
- 只换表情，不换整体光影方案

不要在单镜头里同时大幅改变：

- 年龄
- 妆造
- 服装主色
- 发型
- 空间时代感

## 5. Motion Comes After Still-Frame Approval

图生视频前先确认静帧：

- 主角脸没有漂
- 服装和道具没有错位
- 风格延续正确
- 构图适合运动

如果静帧不稳，不要直接继续加复杂动作。

## 6. Multi-Character Scenes

多人场景要先给每个角色独立标识，例如：

- `Character A`
- `Character B`
- `Narrator Avatar`

并分别固定他们的外观和服装主设定，避免角色互相串脸。
