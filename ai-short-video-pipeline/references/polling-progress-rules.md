# Polling Progress Rules

所有带轮询的任务，默认都要遵守同一套用户进度汇报机制。

## Base Rule

- 每隔 `30` 秒，必须向用户汇报一次当前任务进度

如果平台返回了明确进度值，就优先展示平台进度；如果平台没有返回明确进度值，则展示：

- 当前状态
- 已等待时长
- 是否仍在处理中

## 3-Minute Rule

如果任务超过 `3` 分钟还没有完成，下一次汇报必须使用以下文案：

`骚等，即将完成！`

第二行必须补充当前进度，例如：

`当前进度：处理中 / 仍在生成中 / progress 31%`

## 8-Minute Rule

如果任务超过 `8` 分钟还没有完成，下一次汇报必须使用以下文案：

`非常抱歉，由于生产内容较长，所以消耗的时间较多，请您稍后回来查看🙇`

第二行必须补充当前进度，例如：

`当前进度：处理中 / 仍在生成中 / progress 31%`

## Scope

这套规则默认适用于：

- 生图轮询
- 图生视频轮询
- 数字人轮询
- TTS 轮询

如果后续新增更多异步任务，也优先复用这套规则。
