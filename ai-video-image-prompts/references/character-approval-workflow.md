# Character Approval Workflow

这个流程用于在批量生图前先锁定角色。

## 1. Required Questions

拿到文案后，先问：

- 要生成多少张图片
- 图片比例是什么
- 输出语言是什么
- 目标风格是什么

这一步没完成，不要直接生成整批提示词。

## 2. Draft A Character Proposal

先给用户一版角色方案，而不是直接给 30 条提示词。

角色方案至少包含：

- 角色身份
- 性别表现
- 年龄段
- 脸部特征
- 发型和发色
- 服装
- 气质
- 风格适配说明

## 3. Approval Gate

角色必须过用户确认：

- 用户不满意：继续修改
- 用户提供参考图：以参考图为最高优先级重做
- 用户满意：进入三视图阶段

## 4. Three-View Handoff

角色确认通过后，先生成角色三视图提示词：

- 正视图
- 侧视图
- 背视图

三视图的目标不是做剧情图，而是锁定：

- 脸部结构
- 发型
- 服装
- 配色
- 身材比例
- 角色标志性元素

## 5. Prompt Batch Comes Last

只有完成上面步骤后，才开始生成整批图片提示词。

整批提示词里，每一句都应继承：

- 同一角色
- 同一风格
- 同一比例
- 同一语言要求
