# Cyberpunk Template 01

这是可迁移的赛博朋克风格模板，不绑定具体人物或具体物体。

## Style Block

`赛博朋克风格，偏动漫化和风格化表达，避免写实风格，蓝紫青粉霓虹光，未来都市科技感，电子屏与全息 UI 氛围，金属与玻璃材质，强透视，空间纵深明显，画面通透，镜头感强`

## Color Tendency

- 蓝紫
- 青色
- 电光粉
- 局部霓虹橙高亮

## Lighting Tendency

- 夜景霓虹边缘光
- 阳光明媚的高通透白天光效
- 高对比但主体清晰
- 冷暖混合环境光
- 局部发光屏幕照亮主体

## Material Tendency

- 金属
- 玻璃
- 发光电子面板
- 全息界面
- 数据流
- 自然植被与科技材质共存

## Environment Tendency

- 未来都市
- 科技办公空间
- 数字控制台
- 霓虹走廊
- 企业级工作台界面
- 赛博朋克自然环境
- 阳光明媚的高科技室外场景
- 自然植被与未来建筑结合的开放空间

## Composition Tendency

- 强透视
- 前中后景分层清晰
- 人物镜头优先半身或中近景
- 场景镜头优先纵深和空间感

## Subject Consistency Template

优先复用通用模板：

- [subject-consistency-template-01.md](./subject-consistency-template-01.md)

赛博朋克场景下的默认继承写法：

`人物主体严格保持与参考图一致的样貌、发型、穿着和整体打扮，不改变人物脸部识别特征；常见物体主体严格保持与参考图一致的造型、结构、颜色和识别特征，不改变主体本身设定，只允许做环境适配和风格统一。`

## Brand Element Rule

如果文案里明确出现大众熟知品牌或平台，可直接作为正式画面元素写入，例如：

- 微信 / 企业微信
- 华为 / 华为云
- 腾讯云
- Coze
- Kimi
- MiniMax

默认把这些元素作为：

- 场景识别元素
- 平台界面元素
- 品牌视觉元素

不默认把它们当作主体预览对象。

## Negative Constraints

`无文字，无水印，无杂乱排版，避免低清晰度，避免主体跑形，避免过暗，避免霓虹过曝，避免人物脸部被灯光遮死，避免人物机甲化，避免把常见动物或物体改成完全不同的另一种生物或结构，避免写实照片风，避免纪实真人感过强。`

## Prompt Structure Template

每条正式提示词默认按这个顺序组织，并优先贴近 Google 官方 Imagen 提示词方法：

1. 主体
2. 背景与环境
3. 风格
4. 镜头 / 构图
5. 光线 / 时间氛围
6. 材质 / 细节
7. 比例
8. 主体一致性块
9. 负面约束

## Multi-Subject Narrative Rule

这套“主体 -> 背景与环境 -> 风格”结构，优先适用于：

- 单主体叙事镜头
- 单主体场景镜头
- 主次关系非常明确的双主体镜头

如果是正式分镜图中的多主体对照镜头，不要把多个主体的三视图参考图同时作为强参考挂进去。

默认规则：

- 正式分镜图只给主主体挂强参考图
- 次主体优先用文字约束、品牌识别元素、工作区标识或弱参考描述
- 如果必须出现多个主体，也要明确主次，不要让多个主体同时争夺主画面
- 不要把“对照关系”写成角色展示页或设定板

风险提示：

- 多张三视图参考图 + 多主体都要求严格一致 + 对照式构图
- 很容易让模型误判成角色设定图、展示页、三视图页
- 而不是自然的叙事镜头

```text
常见物体主体严格保持与参考图一致的造型、结构、颜色和识别特征，不改变主体本身设定；

```

## Official Alignment Note

参考 Google Cloud Imagen 官方提示词指南时，优先记住这三个核心块：

1. 主体
2. 背景与环境
3. 风格

在这三块明确后，再继续补：

- 镜头距离
- 光线
- 材质
- 质量修饰词
- 负面约束

不要把提示词写成只有风格词和抽象氛围词，而缺少明确主体与环境。
