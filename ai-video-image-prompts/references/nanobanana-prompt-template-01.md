# Nanobanana Prompt Template 01

这个模板用于 `nanobanana-2`、`Imagen` 一类更偏结构化执行的图像模型，核心目标是：

- 让主体、环境、风格之间的层级更清楚
- 提高批量生图的一致性
- 让同一批次镜头更容易复用模板
- 更适合“文案 -> 多镜头 -> 批量执行”的工作流

## Core Writing Principle

更适合 `nanobanana-2` 的提示词写法，不是自然长句优先，而是：

1. 先明确主体
2. 再明确背景与环境
3. 再明确风格和修饰

然后再补：

- 镜头
- 光线
- 材质
- 比例
- 一致性约束
- 负面约束

## Recommended Structure

每条正式提示词优先按这个顺序写：

1. 主体
2. 背景与环境
3. 风格
4. 镜头 / 构图
5. 光线 / 时间氛围
6. 材质 / 细节
7. 比例
8. 主体一致性块
9. 负面约束

## Writing Style

推荐写法：

- 分块清晰
- 一层一层告诉模型画什么
- 主体和背景不要混写成一坨
- 一致性约束单独写
- 批量镜头尽量保持同一结构

不推荐写法：

- 太像口语叙事
- 主体和环境关系不清楚
- 风格词、环境词、动作词堆在一起没有层次

## Example Skeleton

```text
[主体]；
[背景 / 环境]；
[风格]；
[镜头 / 构图]；
[光线 / 时间]；
[材质 / 细节]；
[比例]；
[主体一致性约束]；
[负面约束]。
```

## Example

```text
主讲人物半身站在未来控制台前；
背景是有清晰 GitHub 趋势榜和 Hermes agent 项目标识的赛博科技空间；
赛博朋克风格，偏动漫化和风格化表达，避免写实风格；
中近景；
夜景霓虹边缘光；
金属、玻璃、电子屏、全息界面材质；
16:9；
人物主体严格保持与参考图一致的样貌、发型、穿着和整体打扮；
无多余文字，无水印，无杂乱排版。
```

## Brand And Landmark Rule

如果品牌、平台、地标、社会常见标识需要明确识别，应在结构块里直接写清楚，例如：

- 背景中官方 logo 清晰可见
- 工作区标识清晰可见
- 品牌识别明确
- 地标特征明确可识别

不要只写成抽象氛围。

## Subject Consistency Rule

`nanobanana-2` 更适合明确的一致性块，推荐单独写，例如：

```text
人物主体严格保持与参考图一致的样貌、发型、穿着和整体打扮，不改变人物脸部识别特征；
常见物体主体严格保持与参考图一致的造型、结构、颜色和识别特征，不改变主体本身设定。
```

如果是正式叙事镜头中的多主体场景，默认规则：

- 主主体挂强参考
- 次主体用文字约束或品牌识别描述
- 不要把多个主体的三视图强参考同时塞进一条正式镜头任务里

## Negative Constraints

默认可直接复用：

- 无文字
- 无水印
- 无杂乱排版
- 避免写实风格
- 避免主体跑形
- 避免多余肢体
- 避免过暗
- 避免过曝

## Best Use Cases

这套模板更适合：

- 批量镜头任务
- 多张图的一致性控制
- 主体和环境都需要结构化拆开写的任务
- 文案直接转多镜头提示词
