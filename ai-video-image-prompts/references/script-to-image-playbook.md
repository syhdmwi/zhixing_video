# Script To Image Playbook

这个参考文件定义“文案如何拆成一组图片提示词”。

## 1. Priority Of Visual Beats

优先给这些内容独立画面：

- 开头钩子
- 明确问题
- 用户痛点
- 关键数据
- 强对比
- 解决方案
- 情绪转折
- 收尾 CTA

## 2. Good Prompt Beats

适合单独成图的内容通常包括：

- 人物动作明显
- 信息点清晰
- 情绪变化强
- 能形成具体场景
- 能形成前后对比

## 3. Weak Prompt Beats

不适合直接成图，除非做了具象化改写：

- 纯抽象概念
- 纯逻辑连接词
- 只有“因此、所以、但是”的句子
- 重复解释同一件事的句子

## 4. How To Visualize Abstract Claims

把抽象句子改写成具象场景，例如：

- “门槛降低了” -> 用户轻松点几下鼠标完成接入
- “主动推送消息” -> AI 在企业微信里主动弹出通知和任务卡片
- “看到了曙光” -> 角色面对亮起的屏幕，气氛从困惑转为兴奋

## 5. Batch Quality Checks

一批提示词完成后，至少检查：

- 是否大量重复同一种构图
- 是否大量重复同一种动作
- 是否每条都能映射回文案内容
- 是否开头和结尾有独立强化镜头
- 是否主角一致
- 是否重复出现的动物、吉祥物、产品、关键物件保持一致
- 是否风格一致
- 纯场景画面是否至少占三分之一
