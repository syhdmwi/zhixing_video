# Subject Consistency Rules

这里的“一致性”不只针对人物，也针对所有会重复出现的主体。

## What Counts As A Recurring Subject

以下都应被视为可能需要锁定的一致主体：

- 主讲人物
- 重复出现的动物
- 红色卡通龙虾这类吉祥物或 IP 形象
- 品牌 mascot
- 重复出现的产品
- 重复出现的设备、工具、关键道具
- 同一体系下反复出现的一组角色，例如多个 AI 虚拟小人、不同岗位的数字员工

## Screening Rule

不是所有重复提到的名词都必须先做主体预览。默认只有“需要稳定外观、且会多次直接入镜”的对象，才进入主体锁定流程。

通常应进入主体锁定流程的：

- 主讲人物
- 重复出现的动物、吉祥物、IP 形象
- 有明确视觉造型要求、且会多次出现的产品或关键道具

通常不需要单独锁定的：

- 平台名、品牌名、服务商名
- 通用软件界面
- 抽象概念词
- 只在个别画面充当背景信息的元素

但这不等于这些元素不能进入正式提示词。只要文案里明确出现了大众熟知的品牌、logo、地点或标志性物品，就应允许直接把它们写成正式画面元素，例如：

- 微信、企业微信
- 华为、华为云
- Coze
- 腾讯云
- 大众熟知的地标或设备外观

这类元素默认作为场景识别信息、界面元素或品牌识别元素使用，不必先单独做主体预览。

如果某个品牌、项目或平台最核心的识别方式是固定 mascot、IP 角色、吉祥物或动物形象，则不要只把它当作抽象品牌元素。

例如：

- OpenClaw 的小龙虾形象
- 某些平台固定的机器人或动物 mascot

这类对象应优先按“可复用主体”处理：

- 会反复入镜时进入主体锁定流程
- 不反复入镜时也应在正式提示词里明确写出其形象特征

## Publicly Known Subject Rule

如果文案中的主要主体本身在现实世界、公开传播或行业语境里已经有稳定、明显、被广泛认知的现成形象，应默认优先参考它已有的公认形象去制作。

例如：

- 已有固定吉祥物的项目
- 已有固定动物或 mascot 形象的平台
- 公众熟知且外观稳定的角色或标志性主体

默认处理方式：

1. 先判断它是否已经有公认的视觉形象
2. 如果有，优先沿用该主体已有形象
3. 不要无故重新设计成完全不同的新角色
4. 如果用户提供了更具体的参考图，则以用户参考图为最高优先级

## Common Brand And Landmark Rule

对于以下这类大众熟知对象，默认允许直接识别并写入正式提示词：

- 热门品牌
- 热门平台
- 城市地标
- 景区地标
- 社会常见标识
- 服装品牌
- 汽车品牌
- 设备品牌

如果用户希望观众一眼认出该对象，优先使用更硬的写法：

- `官方logo清晰可见`
- `品牌识别明确`
- `工作区标识清晰可见`
- `地标特征明确可识别`

不要只写成：

- 品牌氛围
- 城市感觉
- 平台元素
- 地标风格

因为这类松写法容易导致模型只画出泛化图标或抽象环境，而不是用户真正想要的可识别对象。

## New Or Ambiguous Object Rule

如果对象满足以下任一情况，应优先请求参考图、参考链接或更明确的识别说明，而不是硬猜：

- 品牌很新
- 品牌较小众
- 对象较垂直
- 名称有歧义
- 视觉标识不稳定
- 你无法高把握度判断它最重要的外观特征

默认处理顺序：

1. 大众熟知对象 -> 直接按明确识别版写入提示词
2. 新的、冷门的、易歧义对象 -> 先补参考，再进入正式提示词

## Subject Family Rule

如果文案里反复出现的不是“一个主体”，而是“同体系的一组角色”，应按主体族群处理。

例如：

- AI虚拟小人团队
- 不同岗位的数字员工
- 同一家族的机器人小队

这类主体需要同时满足两件事：

- 保持统一设计语言
- 每个成员又能被区分

默认区分方式：

- 不同颜色
- 不同胸牌或头部标识
- 不同岗位工具
- 不同职责配件

不要把整组角色画成完全相同的复制体。

执行层规则：

- 如果需要为多个角色做三视图，可以并行提交多个任务
- 但每个任务必须只画一个主体
- 不允许在同一个三视图任务里混入多个角色

正式分镜图规则：

- 多主体正式分镜图，不要同时挂多个主体的三视图参考图作为强参考
- 正式分镜图默认只给主主体挂强参考图
- 次主体优先改用文字约束、品牌识别元素、工作区标识或场景描述来控制
- 如果一张图里同时出现多个主体，必须明确谁是主主体、谁是次主体
- 不要让正式分镜图退化成角色展示页、设定板、三视图页

## Environment Variety Rule

正式提示词批次里，不要把所有镜头都写成同一种环境。

默认应主动穿插：

- 白天场景
- 夜晚场景
- 室内场景
- 室外场景

即使整条内容围绕同一个核心世界观，也要保留环境变化、空间变化和时间氛围变化，避免视觉疲劳。

## What Must Stay Stable

如果某个主体会在多个镜头中反复出现，默认不应无理由改变：

- 颜色
- 轮廓
- 体型或比例
- 表情体系
- 材质感
- 识别特征
- 核心配件或装饰

如果是主体族群，则稳定项还包括：

- 同一体型体系
- 同一材质体系
- 同一世界观来源

差异项则应稳定保持：

- 每个角色自己的颜色标记
- 每个角色自己的岗位标识
- 每个角色自己的工具或职责配件

对于人物主体，正式分镜图里还应默认保持构图策略稳定：

- 优先半身照
- 不默认切成全身照
- 让脸部和上半身保持更高识别度

## Example

如果文案里反复出现“红色卡通趣味龙虾 AI”，那它在整批图中应保持：

- 一样的红色主色
- 一样的卡通造型语言
- 一样的趣味感和表情方向
- 一样的识别度

不能在某些镜头里变成写实龙虾，在另一些镜头里变成机械硬核龙虾，除非用户明确要求变化。

## Prompt Writing Rule

当存在重复主体时，提示词里应明确继承该主体设定，而不是只在第一条提一次。

## Reference Fallback Rule

如果某个重复主体连续多次预览都不能让用户满意，应主动请求参考图或参考链接。参考图优先级高于继续凭文字调整。

## QC Rule

批次完成后，必须检查：

- 同一个主体是否在不同镜头里跑形
- 是否有颜色漂移
- 是否有材质漂移
- 是否有比例或识别特征漂移

如果 QC 已经能明确判断结果不合格，必须先中止继续流转，并提醒用户当前结果需要重生成或修改要求，不能默认继续进入下一步。
