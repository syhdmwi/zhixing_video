# Subject Consistency Template 01

这是可迁移的主体一致性通用模板，不绑定某一个具体人物、动物、吉祥物、产品或道具。

## Use Case

适用于这些情况：

- 换了新文案
- 换了主人物
- 换了高频动物或吉祥物
- 换了常见物体或关键产品
- 但仍然想保持同一套一致性控制方法

## Placeholder Design

使用时，只替换这些占位符：

- `{main_subject}`
- `{main_subject_reference}`
- `{recurring_subject_2}`
- `{recurring_subject_2_reference}`
- `{recurring_subject_3}`
- `{recurring_subject_3_reference}`

如果没有第三个高频主体，就直接删掉对应那一段，不要硬留空位。

## Generic Consistency Block

```text
{main_subject} 严格保持与参考图一致的样貌、发型、穿着、整体打扮和脸部识别特征，不改变主体本身设定；
{recurring_subject_2} 严格保持与参考图一致的造型、结构、颜色、比例和识别特征，不改变主体本身设定；
{recurring_subject_3} 如存在，也严格保持与参考图一致的核心外观、结构和识别特征。
所有重复主体只允许做场景适配、机位变化、光影适配和风格统一，不允许无故换脸、换造型、换物种、换结构或换识别特征。
```

## Human Rule

如果 `{main_subject}` 是人物，默认保持：

- 脸部识别特征稳定
- 发型稳定
- 穿着稳定
- 年龄感稳定
- 气质稳定

正式分镜图默认：

- 优先半身照
- 优先中近景
- 不默认切成全身照

除非用户明确要求：

- 更换服装
- 更换发型
- 更换年龄感
- 更换人物身份设计

## Animal / Mascot / Object Rule

如果 `{recurring_subject_2}` 或 `{recurring_subject_3}` 是动物、吉祥物、产品、常见物体或关键道具，默认保持：

- 结构不变
- 轮廓不变
- 主色不变
- 比例不变
- 表情体系不乱漂
- 核心装饰和识别部位不乱改

只允许变化：

- 所处环境
- 光影气质
- 镜头角度
- 画面中的位置关系

## Reference Priority Rule

优先级按这个顺序：

1. 用户提供的参考图
2. 用户明确补充的文字设定
3. 已确认的预览图 / 三视图
4. 模型自由发挥

也就是说：

- 有参考图时，参考图优先级最高
- 不要因为风格变化就重画主体本身

## What Is Allowed To Change

允许变化的内容：

- 场景
- 构图
- 机位
- 光影
- 镜头情绪
- 背景中的品牌元素
- 整体风格环境

## What Is Not Allowed To Change

不允许无故变化的内容：

- 人物脸
- 人物发型
- 人物穿着打扮
- 动物或吉祥物的基本结构
- 常见物体的核心外观
- 主体颜色体系
- 主体识别特征

## Prompt Injection Rule

正式提示词里不要只在第一张提一次一致性要求。只要主体在当前镜头出现，就应在当前提示词里继承一致性约束。

## Quick Example

```text
主讲女生角色严格保持与参考图一致的样貌、黑色长发、红色碎花一字肩连衣裙，不改变人物脸、发型和穿着；
红色卡通小龙虾严格保持与参考图一致的造型、结构、颜色和识别特征，不改变主体本身设定。
只允许做场景变化、光影适配、构图变化和风格统一。
```

## Reuse Rule

后面换新文案时，只要：

- 替换主体名字
- 替换参考图来源

就能继续沿用这份模板，不需要重写整套一致性逻辑。
