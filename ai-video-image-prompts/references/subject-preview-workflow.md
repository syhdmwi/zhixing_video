# Subject Preview Workflow

当文案里有会反复出现的人或物时，正式 30 张图之前先做主体清单确认和三视图确认。

## 1. Identify Recurring Subjects

至少找出：

- 重复出现的人
- 重复出现的动物或吉祥物
- 重复出现的产品、设备、关键物件

如果文案里反复出现的是“一组角色”，也要识别出来，不要误判成单一主体。

但先做筛选。默认只有“需要稳定视觉身份且会多次入镜”的主体，才进入预览图阶段。

优先进入预览阶段的通常是：

- 主讲人物
- 重复出现的动物、吉祥物、IP
- 有明确固定造型的产品或关键道具

默认不单独进入预览阶段的通常是：

- 平台名、品牌名、服务商名称
- 普通 UI 页面或图标
- 抽象概念
- 只作为背景信息出现的一次性元素

## 2. Generate Three Views First

在进入正式批次之前，默认先为这些主体生成三视图，给用户直观看。

优先顺序：

- 人物三视图
- 物件或吉祥物三视图

如果同时存在多个主体：

- 每个主体单独生成一张三视图
- 不要把两个或多个主体拼成一张三视图参考图
- 用户应能逐个确认每个主体
- 可以并行提交多个独立三视图任务
- 但每个任务内部只能有一个主体

如果存在主体族群：

- 先给出族群设计方案
- 再分别生成关键角色的独立三视图

例如 AI 虚拟小人团队，默认至少拆成：

- 指挥型
- 开发型
- 分析型

默认要求：

- 白底
- 单主体
- 无剧情背景
- 便于用户直接确认主体外观
- 无任何文字、字母、水印、logo、版式元素

人物三视图默认值：

- 如果用户给了人物参考图，直接以参考图为最高优先级
- 如果用户没给人物参考图，主讲人物默认先按动漫人物形象生成
- 如果用户明确要求写实、纪实、真人感，再切换到非动漫方向

如果用户给的是已经合格的人物三视图：

- 直接把这张三视图作为主讲人物锚点
- 不要重复生成主讲人物三视图
- 不要把旧的人物结果拿来替代用户刚给的新参考

但不要把“已有三视图”简单理解成一定适合直接进入正式批次。拿到用户三视图后，先判断它是否适合作为模型可读的主体参考卡。

适合直接使用的三视图通常满足：

- 正面、侧面、背面清楚分开
- 背景简单，最好是白底或浅色纯底
- 姿态稳定，手臂不遮挡脸、发型、衣服关键结构
- 衣服结构、发型轮廓、体型比例清楚
- 没有复杂动作、强背景色、复杂装饰干扰

如果用户提供的三视图存在以下问题，应提醒用户“这张可以用，但可能导致重影、角度融合或姿态干扰”，并建议先生成一张更干净的主体参考卡：

- 动作幅度过大
- 手臂遮挡关键身体结构
- 强色背景容易被模型带入正式图
- 三个角度间距太近或边界不清楚
- 带有太多文字、标注、背景元素

主体参考卡的目标不是重新设计人物，而是把用户给的人物整理成更适合模型读取的参考资料。它应保留原人物的脸型、五官、发型、发量、服装、配饰和整体气质，只优化展示方式。

更推荐的主体参考卡要求：

- 白底或浅色纯背景
- 正面、侧面、背面分栏清楚
- 三个角度留足间距
- 自然站姿或轻微 T pose
- 手臂尽量自然下垂或轻微张开
- 不遮挡脸、衣服、发型关键结构
- 不做剧情背景
- 不添加复杂文字、logo、水印

如果用户同时给了风格参考图和人物三视图：

- 风格参考图只用于定义整体视觉语言
- 人物三视图只用于锁定主讲人物外观
- 不能把风格参考图中的人物形象挪用到主讲人物身上，除非用户明确要求

## 3. Approval Gate

用户必须先确认：

- 主体外观是否满意
- 配色是否满意
- 造型是否满意
- 气质或材质是否满意

展示完三视图后，固定给用户两个选择：

- 需要修改主体请输入 `1`，并直接说明“修改哪个主体 + 修改内容”，也可以上传参考图
- 不需要修改请输入 `2`

如果用户选择修改：

- 可以直接口头描述修改要求
- 也可以上传参考图

如果不满意，先修改三视图，不进入正式批次。

如果同一主体连续多次修改仍不满意，主动询问用户是否有参考图、参考链接或更接近的视觉样例。不要无限继续盲目生成。

如果三视图结果本身就明显不合格，也必须主动拦下，不要继续往下走。典型情况包括：

- 没有做到单主体
- 多个主体混在一张三视图里
- 出现文字、水印、logo
- 角色跑形或与已确认参考明显不一致

这时应先明确告诉用户“当前结果不合格，需要重生成”，然后等待用户决定。

## 4. Use Approved Three Views As References

用户满意后，当前这组三视图就直接作为正式批次的参考图使用，不再默认重复生成一轮新的三视图。

## 5. Final Batch Comes Last

只有在主体清单和三视图都确认后，才进入正式提示词生成。

正式批次里，如果镜头包含人物，默认应采用半身构图，不要默认做全身站立图。

正式批次里的多主体镜头，还要额外遵守：

- 不要把多个主体的三视图参考图同时作为强参考挂到同一张正式分镜图里
- 正式分镜图默认只让主主体使用强参考图
- 次主体用文字描述、品牌识别元素、工作区标识或弱参考控制
- 避免把叙事镜头写成角色展示页、设定板或三视图页

如果正式分镜图使用人物三视图或主体参考卡作为参考，提示词里必须明确说明：

- 参考图是人物参考卡，不是最终画面内容
- 只用于保持人物身份一致性
- 不要复刻三视图版式
- 不要生成多角度展示页
- 不要把正面、侧面、背面融合到同一个人物身体里
- 最终画面只生成一个符合当前镜头角度的人物

注意：

- 这条“半身构图”规则针对正式分镜图
- 三视图为了锁定角色外观，可以继续按完整展示需求执行
