# 用户使用说明

这套工作流包是给 AI 编程工具和智能体工具用的 AI 短视频助手。

它不是一个单独的软件界面，而是一套可以让 AI 助手按固定流程帮你完成短视频生产的能力包。

你可以在 Codex、OpenClaw、Trae、Claude Code 等工具里使用。安装或导入完成后，直接在对话里用自然语言开始。

如果你使用的不是 Codex，请先看：

- [跨工具使用说明](./跨工具使用说明.md)

如果你想照着步骤操作，请看：

- [SOP：如何使用知行视频skill](./SOP-如何使用知行视频skill.md)

## 它能帮你做什么

这套工作流主要适合 4 类任务：

1. 把一段文案变成一组图片提示词
2. 根据提示词批量生成图片
3. 把图片继续生成视频
4. 上传音频和人物视频，生成数字人
5. 把满意的主讲人、风格和提示词规则保存成模板，下次直接套用

它也可以把这些步骤串起来，从文案一路做到视频。

## 安装后第一句话怎么说

如果你不知道怎么开始，直接说：

```text
这是我的文案，帮我从头开始做。
```

如果你已经知道自己要做哪一步，可以用下面这些句子。

## 推荐唤醒词

这套工作流可以统一叫：

```text
知行视频skill
```

推荐这样唤醒：

```text
使用知行视频skill，帮我从文案开始做视频。
```

或者：

```text
用知行视频skill帮我制作视频。
```

如果你只输入：

```text
知行视频skill
```

系统应该先问你想做哪一步，而不是让你自己找内部模块。

如果你中途忘了现在走到哪一步，或者换到 Trae、OpenClaw、Claude Code 后流程接不上，直接说：

```text
使用知行视频skill，帮我恢复当前流程状态。
```

系统应该先告诉你当前阶段、已完成内容和下一步可回复什么，然后再继续执行。

## 常用开口句

### 1. 从文案开始做图片

```text
这是我的文案，先带我走图片流程。
```

然后把文案粘贴进去。

### 2. 从文案一路做到视频

```text
这是文案，帮我从图片一路做到视频。
```

### 3. 已经有图片，继续做视频

```text
这些图片已经确认，帮我进入视频生成。
```

然后上传图片或提供图片链接。

### 4. 做数字人

```text
我要做数字人，我上传音频和人物视频给你。
```

然后上传音频和正面人物视频。

### 5. 只生成图片提示词，不真实生图

```text
这是我的文案，只帮我生成图片提示词，先不要生图。
```

### 6. 测试整套流程

```text
我想测试这套短视频流程，你一步一步带我走。
```

### 7. 套用已有模板

```text
使用知行视频skill，套用「模板名称」帮我做这篇新文案。
```

## 运行时会怎么问你

正常图片流程大概会这样走：

1. 你发文案
2. 系统问你选择风格方式
3. 系统问你选择图片比例
4. 系统问你选择生图模型
5. 系统识别重复出现的人物、角色、品牌、物体
6. 你确认主体清单
7. 系统生成主体参考图或三视图
8. 你确认主体是否满意
9. 系统展示正式图片提示词
10. 你确认后再开始生图
11. 生图完成后，系统展示全部图片
12. 你决定是否修改图片
13. 不需要修改后再进入视频生成

你不用一次性把所有要求说清楚，系统会一步一步问。

如果你只回复 `1`、`2`、`全部生图`、`进入视频生成` 这类短指令，系统应该先判断当前处在哪个阶段，再解释你的回复，不能跨阶段乱跳。

## 用户需要准备什么

最少准备其中一个就可以开始：

- 文案
- 图片
- 音频
- 人物视频

如果你想效果更稳定，可以额外准备：

- 主讲人照片
- 主讲人三视图
- 角色参考图
- 风格参考图

## 风格怎么说

你可以直接描述：

```text
我想要赛博朋克、半 3D 二次元、短视频封面风格。
```

也可以上传参考图，并说明：

```text
风格参考我上传的图，只参考风格，不参考人物。
```

如果你希望人物也参考图片，要明确说：

```text
人物也参考我上传的图。
```

## 图片生成后怎么继续

图片生成完成后，系统会问你：

```text
是否满意，有无需要修改的图片？
如果有请输入第几张，并把修改要求一起输入；
如果不需要修改，输入 2 进入视频生成。
```

你可以这样回复：

```text
第 3 张，小龙虾太机械了，改成更可爱。
```

或者：

```text
2
```

## 怎么保存模板

当一批图片或视频效果满意后，可以保存成模板。

模板默认保存：

- 主讲人形象
- 主讲人三视图
- 画面风格
- 提示词规则
- 负面约束
- 默认比例和模型

模板默认不保存：

- 每篇文案临时出现的重复主体
- 用户没有选择的正式图

如果系统问：

```text
是否要把这次的主讲人、风格和提示词规则保存为模板？
```

你可以直接输入模板名称：

```text
赛博女主持模板
```

如果系统问是否保存满意样例图，可以输入图片编号：

```text
1、3、8
```

如果不保存样例图，输入：

```text
2
```

## 视频生成怎么选

进入视频阶段后，会有两个选择：

```text
1 图转镜头移动视频模式
2 图生视频模式
```

如果你只是想让图片像镜头一样平稳移动，选 `1`。

如果你想让 AI 根据图片生成动态视频，选 `2`。

## 数字人怎么用

直接说：

```text
我要做数字人，我上传音频和人物视频给你。
```

然后上传：

- 一段音频
- 一段正面人物视频

如果系统已经配置了对象存储，你不需要自己处理公网链接，系统会自动转存后提交。

## 常见误区

- 不需要先学习每个目录或模块的名字。
- 不需要一开始就配置所有 API key，只测哪段就配哪段。
- 不需要自己写完整提示词，系统会先展示给你确认。
- 不满意图片时，不需要重跑整批，可以指定第几张修改。
- 视频默认只交付每个镜头的视频，不会自动拼成完整总片，除非你明确要求。

## 最推荐的新手测试方式

第一次测试建议这样说：

```text
我想测试这套短视频流程，你一步一步带我走。

文案是：
[把你的文案粘贴在这里]
```
