# 跨工具使用说明

这套包最初按 Codex skills 的目录结构整理，但核心内容不是 Codex 专用。

它本质上是一套 AI 短视频生产工作流规则，可以迁移到：

- Codex
- OpenClaw
- Trae
- Claude Code
- 其它支持项目规则、上下文文档或自定义工作流的 AI 编程工具

## 核心思路

不同工具的“技能系统”名字可能不同。

有的叫 skill，有的叫 rules，有的叫 memory，有的叫 project instructions，有的只是把 Markdown 文件放进项目上下文。

只要工具能读取这些说明，并按说明执行，就可以使用这套工作流。

## 通用唤醒规则

建议在 Codex、OpenClaw、Trae、Claude Code 等工具里都加入这条规则：

```text
当用户提到“知行视频skill”“知行视频 skill”“知行视频工作流”，或说“使用知行视频skill帮我制作视频/图片/数字人”时，优先读取 ai-short-video-pipeline/SKILL.md，并按其中流程引导用户。
```

如果用户只说了唤醒词，没有给文案、图片、音频或人物视频，应先问：

```text
你想用知行视频skill做哪一步？
1. 从文案开始做图片
2. 从图片继续做视频
3. 做数字人
4. 从文案一路跑完整流程
```

## 在 Codex 里使用

把这些模块目录复制到 Codex 的 skills 目录：

```text
$CODEX_HOME/skills/
```

然后重启或刷新 Codex。

使用时直接说：

```text
这是我的文案，帮我从头开始做。
```

## 在 OpenClaw 里使用

推荐方式：

1. 把整个项目文件夹加入 OpenClaw 工作区
2. 把 [ai-short-video-pipeline/SKILL.md](./ai-short-video-pipeline/SKILL.md) 设置为总控规则或主上下文
3. 把 [用户使用说明](./用户使用说明.md) 发给使用者作为入口文档
4. 让 OpenClaw 按目录里的模块说明调用对应脚本或流程

建议给 OpenClaw 的开场指令：

```text
请读取这个项目中的 ai-short-video-pipeline/SKILL.md，把它作为 AI 短视频工作流总控规则。
当用户提到“知行视频skill”时，优先唤醒这个工作流。
用户发文案后，按用户使用说明里的流程一步一步引导。
```

## 在 Trae 里使用

推荐方式：

1. 用 Trae 打开整个项目文件夹
2. 把 [ai-short-video-pipeline/SKILL.md](./ai-short-video-pipeline/SKILL.md) 和 [用户使用说明](./用户使用说明.md) 加入项目上下文
3. 如果 Trae 支持项目规则，把总控规则写入项目规则
4. 让 Trae 优先按总控规则判断该调用哪个子模块

建议给 Trae 的开场指令：

```text
请把当前项目当作 AI 短视频工作流包使用。
优先阅读 用户使用说明.md 和 ai-short-video-pipeline/SKILL.md。
当用户提到“知行视频skill”时，优先进入这个工作流。
当我输入文案时，请按流程引导我完成图片、视频或数字人生成。
```

## 在 Claude Code 里使用

推荐方式：

1. 用 Claude Code 打开整个项目目录
2. 把 [用户使用说明](./用户使用说明.md) 当作使用者入口
3. 把 [ai-short-video-pipeline/SKILL.md](./ai-short-video-pipeline/SKILL.md) 当作主流程规则
4. 需要图片、视频、数字人时，再读取对应子模块的 `SKILL.md`

建议给 Claude Code 的开场指令：

```text
请读取当前项目里的 用户使用说明.md 和 ai-short-video-pipeline/SKILL.md。
之后把这个项目当作 AI 短视频生产工作流包使用。
当用户提到“知行视频skill”时，优先进入这个工作流。
不要先解释内部模块名，先按用户任务一步一步引导。
```

## 通用启动话术

无论你用哪个工具，安装或导入完成后，都可以这样开始：

```text
使用知行视频skill，帮我从文案开始做视频。
```

或者：

```text
这是我的文案，帮我从头开始做。
```

或者：

```text
我想测试这套短视频流程，你一步一步带我走。

文案是：
[把你的文案粘贴在这里]
```

## 如果工具不支持直接执行脚本

有些工具只能读文档，不能直接运行脚本。

这种情况下也能使用：

- 让工具先生成图片提示词
- 再把提示词复制到你自己的生图工具
- 再把生成好的图片交回工具继续做视频提示词或剪辑规划

也就是说，这套包可以分成两种使用方式：

- 自动执行版：工具能运行脚本和 API
- 流程指导版：工具只负责生成提示词、清单、操作步骤

## 最小可用导入

如果你只想快速试用，不想导入全部文件，最少导入这 3 个：

1. [用户使用说明](./用户使用说明.md)
2. [ai-short-video-pipeline/SKILL.md](./ai-short-video-pipeline/SKILL.md)
3. [ai-video-image-prompts/SKILL.md](./ai-video-image-prompts/SKILL.md)

这样至少可以跑通：

- 文案理解
- 风格选择
- 主体识别
- 图片提示词生成

等确认流程有用后，再补充生图、视频、数字人模块。
